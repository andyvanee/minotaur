Minotaur a minimalistic CSS framework

What is it?

Minotaur is a small framework built on top of normalize.css with the goal to make it responsive and customizable.

Minotaur doesn't include a massive widget collection. It only provides a customizable grid, a clean approach to typography, and easy theming of core HTML elements.

Usage

Minotaur written using less.css. Your own less file should look something like the following: 

@import 'minotaur/variables';      // Minotaur grid system variables
@import 'variables';               // your own variables to override the defaults
@import 'minotaur/minotaur';       // Minotaur grid system

... the rest of your styles ...

See the README for more details on integrating it into your project.

Grid

The Minotaur grid avoids many of the magical psuedo-classes and negative margins that are so prevalant in frameworks today. It is symmetrical, nestable and follows the screen-dimensions in an intuitive way.

Minotaur has 3 major breakpoints defining a 4, 8 and 12 column grid. A css class of .span-4 is full width at the smallest sizes, half screen at tablet sizes and one-third width at desktop sizes. This can be customized by adding mid-* and large-* classes to override this behaviour.

Column ordering can be adjusted by adding .right classes to any div that should appear on the right for multi-column layouts, but first in single-column layouts.

Also, the grid can be fluid or fixed-size between these breakpoints depending on the values set for the @eight-column-fluid and @twelve-column-fluid variables.

How does that work?

Minotaur uses padding, rem's (root em's) and box-sizing: border-box to perform it's grid magic. The row is padded with half the gutter width and the columns are padded with the other half.

This makes it much easier to add borders & backgrounds or customize the padding of a column without getting into any high-order math.

Responsive margins and paddings

Minotaur adjusts the padding between and around columns as screen real-estate shrinks. See the @column-padding variables in minotaur/variables.less.

Grid Demo

Main Column

This is a demo of a couple of the features of the grid. This column has the following classes: span-8 mid-5 right. The large layout has 12 columns, and this takes up 8 of them. The mid-size layout has 8 columns, and this would take up all of them if not for the .mid-5 class which causes it to use 5 columns. The small layout has only 4 columns, so .span-8 is interpreted as being equivalent to .span-4. The right class indicates that this column should be floated to the right in multicolumn layouts, instead of left as it would by default.

Typography

Normalize-css

The starting point for typesetting in Minotaur is the excellent and ubiquitous normalize.css.

Embracing the Rem's

Typography in Minotaur uses rem's (root em's) quite liberally to set a consistent pace of line-heights and margins.

The core line-height @ui-line-height is set in em's (1.75em by default) so that it matches whatever content it is applied to.

Headings, on the other hand, use margins and line-heights measured in rem's. The .heading() mixin provided in minotaur-ui/mixins.less takes a font size in rem's and produces a heading with the given font-size, a line height rounded up to the nearest rem, and the margins specified by @ui-margin-top and @ui-margin-bottom.
